# My Engineering Preferences

This document outlines my development style and technology preferences to ensure our collaboration is efficient.

## Core Technologies

- **TypeScript:** Primary language (Angular, Node.js).
- **Go:** For backend services, CLIs, and systems tools.
- **C++:** For performance-critical systems programming.

## Guiding Principles

- **Testing is mandatory:** Use Jest for TypeScript and the standard `testing` package for Go.
- **Conventional Commits:** Adhere to the specification for all commit messages.
- **Idiomatic Code:** Follow established conventions for each language.
- **Immutability:** Prefer immutable data structures, especially in TypeScript.

## Language-Specific Style

- **TypeScript:** Use strict typing, modern language features, and project-level ESLint/Prettier rules.
- **Go:** Write idiomatic code (`gofmt`, [Effective Go](https://go.dev/doc/effective_go)). Handle errors explicitly.
- **C++:** Prefer modern C++ (C++17 or newer) and follow the [C++ Core Guidelines](https://isocpp.github.io/CppCoreGuidelines/CppCoreGuidelines).
